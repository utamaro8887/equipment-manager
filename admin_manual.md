# 備品カルテシステム 管理者向け導入・運用マニュアル

本マニュアルは、システムの新規構築・移譲・保守を行うシステム管理者向けの手順書です。

## 1. システム要件と構成
- **フロントエンド:** HTML / CSS (Bootstrap 5) / 原生JavaScript
- **バックエンド:** Google Apps Script (GAS)
- **データベース:** Google スプレッドシート
- **画像保存先:** Google ドライブ (`Equipment_Images` フォルダ)
- **コード管理:** `clasp` および Git

## 2. 新規環境の構築手順 (ゼロから立ち上げる場合)

### Step 1: データベース（スプレッドシート）の作成
1. Googleドライブで新規スプレッドシートを作成し、URLから「スプレッドシートID」を控えます。
   - `https://docs.google.com/spreadsheets/d/【ここの部分がID】/edit`
2. 以下のシート名でシートを作成し、`spreadsheet_schema.md` に記載されている構造通りに1行目（ヘッダー）を作成します。
   - `T_Assets` (備品台帳)
   - `T_History` (更新・履歴記録)
   - `M_Categories` (カテゴリマスタ: 大分類・中分類・コード)
   - `M_Lists` (プルダウンリスト用のマスタ: 設置フロア・設置場所・報告種別など)

### Step 2: GASプロジェクトの作成とコード反映
1. Node.js がインストールされたPCで、ソースコードのフォルダを開きます。
2. ターミナルで `npx clasp login` を実行し、Googleアカウントで認証します。
3. `npx clasp create --type standalone --rootDir gas` を実行し、GASプロジェクトを作成します。
4. `gas/Code.gs` 内の `SPREADSHEET_ID` の値を、Step 1で控えたIDに書き換えます。
5. `npx clasp push` を実行し、コードをGoogle側にアップロードします。

### Step 3: Webアプリとしてのデプロイ (公開設定)
1. ブラウザでGASのエディタ画面を開きます (`npx clasp open`)。
2. 右上の青い「デプロイ」ボタンから「新しいデプロイ」を選択します。
3. 種類の選択（歯車アイコン）から「ウェブアプリ」を選択します。
4. 以下の設定でデプロイを実行します：
   - アクセスできるユーザー: **「全員」 (Anyone)** 
     ※ GoogleアカウントなしでもQR等からアクセスさせるため必須の設定です。
5. 発行された「ウェブアプリのURL」をコピーし、QRコード作成等に利用します。

### Step 4: Googleドライブの権限承認 (初回のみ必須)
本システムは画像をGoogleドライブに保存するため、管理者のアカウントで実行権限を許可する必要があります。
1. GASエディタ上の関数選択プルダウンから `setupPermissions` を選択します。
2. 「実行」ボタンを押し、表示されるダイアログに従って「権限を確認」→「アカウント選択」→「詳細」→「（プロジェクト名）に移動」→「許可」の順に進めます。
3. 実行完了ログに「認証が正常に完了しました」と出れば、画像アップロードが利用可能になります。
4. Googleドライブ内に自動的に `Equipment_Images` フォルダが作成されます。このフォルダの共有設定を「リンクを知っている全員：閲覧者」にしておくと、誰でも画像が表示できるようになります。

## 3. マスタデータの追加・編集方法
システム内のプルダウンやカテゴリは、スプレッドシート上のマスタシートと動的に連動しています。

- **設置フロア・場所・各種プルダウンの追加:**
  `M_Lists` シートの該当列（ヘッダー名の下）に新しい項目を追記するだけで、アプリ上のプルダウンに即時反映されます。

- **新しいカテゴリ（機材種別）の追加:**
  `M_Categories` シートに、「大分類」「中分類」「カテゴリコード」を追記してください。
  ※ 追加後、新規登録時にここで設定したカテゴリコードをもとに「PC2401-01」のような管理番号が自動採番されます。

## 4. 運用上の注意・トラブルシューティング

- **「ファイルを開くことができません」エラー（/u/3/ 問題）**
  複数GoogleアカウントにログインしているブラウザでGASを開くと、URLに `/u/1/` 等が付与されます。このURLを共有すると他の人はアクセスできません。必ず、デプロイ管理画面からコピーした「純粋なURL形式（`/macros/s/AKfyc.../exec`）」を共有またはQRコード化してください。
- **データ不整合・手動修正**
  万が一アプリから更新できなくなった場合、管理者はスプレッドシートの `T_Assets` を直接編集して修正することが可能です（ID列は絶対に書き換えないようにしてください）。
