# プロジェクト状況・仕様まとめ (2026/03/15)

本プロジェクト「設備QR管理システム」の現状と、中断後の再開に必要な情報をまとめました。

## 1. システム全体像
- **目的**: 院内の備品にQRコードを貼り、スマホでスキャンするだけで「点検・修理履歴の確認」と「現場からの点検報告」を行えるようにする。
- **構成**:
  - **DB**: Google スプレッドシート（「備品マスタ」シートと「履歴データ」シート）
  - **Backend**: Google Apps Script (GAS) ※ `clasp` でローカルから管理
  - **Frontend**: GAS Webアプリ (HTML/CSS/JS)

## 2. 現在の進捗状況
### [x] 環境構築完了
- Node.js および `clasp` による開発環境のセットアップ。
- Google Apps Script API の有効化。
- GASプロジェクト `Equipment_QR_Management` の新規作成。

### [x] 基本機能実装完了
- **データ取得**: スプレッドシートID `1996BJT0IJoHYebMcoQaB0V6JNerrgnlyOCJOtUACT94` から情報を取得。
- **表示**: モバイルに最適化したカード形式のUI。
- **報告機能**: 報告フォームから送信し、スプレッドシートの「履歴データ」へ追記＋マスタの最終点検日を自動更新する。
- **UX改善**: GAS特有の `location.reload()` 不具合（白画面化）を回避し、非同期でデータを再取得する仕組みを導入。

## 3. 現在取り組んでいた内容
- **フィールドの拡張**: 「使用場所」「使用部署」「備考」といった管理用項目の追加。
- **棚卸し機能の検討**: 報告時に「最終所在確認日」を更新することで、自動的に棚卸しログを残す仕様。

## 4. 再開時のステップ
1.  **`spreadsheet_schema.md` の確認**: ユーザーが書き換えたスプレッドシートの列構成（A, B, C...）を確認する。
2.  **`Code.gs` の修正**: 新しい列（location, current_dept等）の読み書きを可能にする。
3.  **`index.html` の修正**:
    - 表示部分に「場所」「部署」「備考」を追加。
    - 報告フォームに「現在の場所」を確認・入力できる項目を追加。
4.  **デプロイ**: `clasp push` 後、GASエディタで「デプロイの更新（新バージョン）」を行う。
