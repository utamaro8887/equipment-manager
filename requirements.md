# 備品カルテシステム 要件定義書 (2026/03/16 更新)

## 1. システムの目的・背景
本システムは、地方独立行政法人 たつの市民病院における備品管理のデジタル化を目的とします。

*   **課題**: 電子カルテ（閉域網）と情報系ネットワークの分離によるデータの二重管理、棚卸しの形骸化。
*   **解決策**: 院内のWi-Fi環境を活用し、ログイン不要でアクセス可能な備品カルテ（GAS Web App）を構築。
*   **目標**: 現場での自己解決促進、情報の集約、棚卸し作業の効率化。

## 2. システム構成
*   **データベース**: Google Spreadsheets (正規化されたマスタ・履歴シート)
*   **バックエンド**: Google Apps Script (GAS) ※ `clasp` によるコード管理
*   **フロントエンド**: GAS HTML Service (Bootstrap 5 を利用したレスポンシブUI)
*   **連携**: LINE WORKS API (Bot) による修理依頼時のプッシュ通知

## 3. 機能要件

### ① 備品識別・認証
*   **QRトークン認証**: `qr_token` パラメータを含むURLをQRコード化し、ログイン不要で該当備品の情報にアクセス。
*   **マニュアル連携**: `manual_url`（GoogleドライブのPDF等）へのワンタップアクセス。

### ② 表示・一覧・報告機能 (単票フォームSPA)
*   **SPA設計**: 画面遷移時の白い画面による待機をなくすため、一覧から詳細画面までを1つのHTMLファイルで実装し、JavaScriptによる即時画面切り替え（SPA）を実現。
*   **一覧表示機能**: 全備品のリスト表示と、名称・管理番号等によるインクリメンタル検索。
*   **単票フォーム機能**: Accessライクな1画面での「新規登録」「確認表示」「情報修正」をシームレスに行う入力インターフェース。
*   **自動採番・管理**: `id`はスプレッドシートの行番号ベースで自動採番。管理番号は「カテゴリ名＋購入西暦下2ケタ＋月2ケタ」で自動計算される。
*   **履歴・ステータス管理**: 稼働中、故障中、予備、廃棄待、廃棄済。点検や移動・廃棄などの日々の報告は、フォーム上の専用報告画面から履歴として追記できる。

### ③ 運用・保守
*   **同時実行制御**: `LockService` によるデータ競合の防止。
*   **棚卸しエビデンス**: 棚卸報告時に所在（部署・場所）を記録することで、会計検査等のエビデンスとする。

## 4. UI/UX方針
*   **iPhone（Safari）最適化**: 現場でのモバイル利用を第一に考えたカードUI。
*   **デジタルデバイド対応**: 直感的なボタン操作、文字の視認性向上。

## 5. 今後の拡張性
*   電子カルテ側の部署マスタとの同期。
*   耐用年数に基づいた更新時期の自動算出。
