# スプレッドシート列構成定義書

## T_Assets シート (Sheet Name: T_Assets)
備品の基本情報を管理するマスターテーブルです。

| 列 | フィールド名 | プログラム用キー | 説明 |
| :--- | :--- | :--- | :--- |
| A | ID | `id` | システム内部で利用する一意の識別子（行番号-1） |
| B | 備品管理番号 | `asset_tag` | カテゴリ＋年月で自動採番 (例: 車2603-01) |
| C | カテゴリコード | `category_code` | 備品の種類（コード: PC, WC等） |
| D | 備品名称 | `name` | 個別の備品名称 |
| E | 型番 | `model_number` | 製品の型番・品番（半角大文字に自動変換） |
| F | 状態 | `status` | 稼働中\|予備\|故障中\|廃棄待\|廃棄済 |
| G | 設置フロア | `floor` | 物理位置（大分類） |
| H | 設置場所 | `location` | 物理位置（小分類） |
| I | メーカー | `manufacturer` | **Manufacturer Name** (追加分・QRアクセスキー跡地) |
| J | 資産区分 | `asset_class` | 備品\|固定資産（会計報告対象） |
| K | 購入年月日 | `purchase_date` | 備品を導入した日 |
| L | 購入金額 | `price` | 備品を導入した金額 |
| M | 説明書リンク | `manual_url` | 取扱説明書(PDF)等のURL |
| N | 備考 | `note` | 備考 |
| O | 購入業者 | `vendor` | 備品を導入した業者 (追加分) |
| P | 入力担当者名 | `input_operator` | 入力担当者名 |
| Q | 耐用年数 | `useful_life` | (任意追加) |
| R | 修理依頼先 | `repair_vendor` | (任意追加) |
| S | 画像URL | `image_url` | 備品の画像URL |

## T_History シート (Sheet Name: T_History)
点検や修理、移動などのイベントを記録する履歴テーブルです。

| 列 | フィールド名 | プログラム用キー | 説明 |
| :--- | :--- | :--- | :--- |
| A | 報告日時 | `timestamp` | メンテナンスや移動が報告された日時 |
| B | 対象備品ID | `asset_id` | T_Assets.id への参照キー |
| C | 報告種別 | `type` | 修理\|移動\|点検\|廃棄\|その他\|棚卸 |
| D | 担当者名 | `operator` | 報告を行った職員名 (自己申告) |
| E | 備考・メモ | `note` | 自由記述（「：なし」はこの項目の内容を表示） |
| F | 画像URL | `image_url` | 報告時の画像URL |

## マスタシート (M_Depts / M_Categories)
将来的な正規化（名称変更への耐性強化）のためのマスタ構成です。

### M_Depts シート (部署マスタ)
| 列 | フィールド名 | 説明 | 
| :--- | :--- | :--- |
| A | 部署ID | システム内部管理用のID |
| B | 部署コード | 院内の既存コード (例: D001) |
| C | 部署名 | 正式名称 (例: 事務局) |
| D | 表示順 | 選択肢の並び順 |
| E | 有効フラグ | 1:有効, 0:無効 |

### M_Categories シート (カテゴリマスタ)
| 列 | フィールド名 | 説明 |
| :--- | :--- | :--- |
| A | カテゴリID | システム内部管理用のID（連番） |
| B | 大分類 (L1) | 階層1 (例: IT機器, 施設・什器) |
| C | 中分類 (L2) | 階層2 (例: パソコン, プリンタ) |
| D | カテゴリコード | 管理番号の接頭辞に使用 (例: PC, WC) |
| E | 備考 | 補足情報 |

### M_Lists シート (簡易リスト)
現在アプリのプルダウン生成に使用している、1行目に項目名を持つシンプルなリストです。
| フロア | 場所 |
| :--- | :--- |
| (リスト項目) | ... |
