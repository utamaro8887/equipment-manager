# ユーザー実施事項 (ToDoリスト)

システムの仕様変更およびアプリの更新に伴い、スプレッドシート側で以下の対応をお願いいたします。

## 1. T_Assets シート（備品台帳）の列の並べ替え・作成
アプリが正しくデータを読み書きできるよう、以下の順番・ヘッダー名で列を整えてください（既存データがある場合は移動をお願いします）。

| 列 | ヘッダー名（推奨） |
| :--- | :--- |
| A | ID |
| B | 備品管理番号 |
| C | カテゴリコード |
| D | 備品名称 |
| E | 型番 |
| F | 状態 |
| G | 設置フロア |
| H | 設置場所 |
| I | 資産区分 |
| J | QRアクセスキー |
| K | 購入年月日 |
| L | 購入金額 |
| M | 説明書リンク |
| N | 備考 |
| O | 購入業者 |
| P | 入力担当者名 |
| Q | 耐用年数 |
| R | 修理依頼先 |

## 2. T_History シート（履歴台帳）のヘッダー・選択肢更新
- **C列**: 「報告種別」の選択肢を **「修理」「移動」「点検」「廃棄」「その他」「棚卸」** にしてください。
- **D列**: 名称を 「担当者名」 → **「登録者名」** に変更してください（※内部キーは `operator` で維持）。

## 4. M_Categories シート（カテゴリマスタ）の作成
新規に `M_Categories` シートを作成し、以下の構成でデータを入力してください。
このマスタにより、アプリ上で「大分類 → 中分類」の連動プルダウンが有効になります。

| 列 | ヘッダー名 | 説明 |
| :--- | :--- | :--- |
| A | カテゴリID | 管理用の連番 |
| B | 大分類 (L1) | IT機器、施設・什器 など |
| C | 中分類 (L2) | パソコン、プリンタ、車椅子関連 など |
| D | カテゴリコード | PC, WC, JK などの英数字コード |
| E | 備考 | 補足情報 |

## 5. M_Lists シート（その他入力支援リスト）の作成
既存の `M_Lists` シートの「カテゴリ」「メーカー」「部署」列は使用しなくなりました。必要に応じて列を削除し、以下のヘッダーでリストを作成・維持してください。

- **A列**: フロア
- **B列**: 場所

## 4. プログラムのデプロイ
もしご自身で `clasp` などを用いてデプロイされる場合は、以下のコマンドを実行してください。
```powershell
npx clasp push
```

## 5. 動作確認
上記が完了したら、アプリをリロードして以下の挙動を確認してください。
- 設置フロアを選択した際、設置場所の候補が絞り込まれるか（マスタに「3Fナースステーション」等の文字が含まれる場合）。
- 購入金額を10万円以上にした際、資産区分のアラートが出るか。
- 入力担当者名が正しく保存されるか。
